# SDP
SmartDialog Processor (SDP) - 项目目标说明书

一、核心业务目标

1. 核心功能目标

• 对话内容智能识别：实现90%+准确率的对话文本OCR识别

• AI个性化响应：基于"旮旯给木"基础模板，生成3-5个差异化选项

• 用户偏好学习：通过选择统计，7天内建立用户语言风格画像（幽默/高冷/正式等）

• 模板动态优化：根据用户选择反馈，每周自动优化模板库，提升选项匹配度

2. 用户体验目标

• 响应时间：从截图到显示AI选项 ≤ 3秒（端到端）

• 操作便捷性：一键截图+框选，3步内完成完整流程

• 界面友好度：新手用户5分钟内掌握全部操作

• 个性化程度：使用2周后，AI选项符合用户偏好的概率 ≥ 70%

二、技术性能指标

1. 系统性能

• 并发处理：支持1000+用户同时在线使用

• API响应：后端API平均响应时间 < 500ms

• 数据存储：用户对话记录保存6个月，统计数据分析实时更新

• 可用性：系统整体可用性 ≥ 99.5%（SLA）

2. AI处理质量

• 选项多样性：每次生成3-5个语义不同但合理的选项

• 模板覆盖：基础模板库包含20+种对话场景模板

• 优化速度：用户选择数据积累100条后，系统开始自动优化模板

• 准确率提升：每月模板优化使选项点击率提升5-10%

三、业务价值目标

1. 用户价值

• 效率提升：相比手动输入回复，节省50%+的对话时间

• 沟通质量：提供更符合场景的回复建议，提升沟通效果

• 个性化体验：系统越用越懂用户，形成个性化对话助手

2. 数据价值

• 用户画像：积累用户语言风格、对话偏好数据

• 场景分析：识别高频对话场景和痛点

• 模板优化：形成可商业化的对话模板库

3. 扩展性目标

• 多平台支持：第一阶段桌面端，第二阶段移动端适配

• 多语言扩展：支持中英文，预留其他语言接口

• API开放：后期可开放部分API给第三方开发者

四、阶段性里程碑

MVP阶段（1-2个月）

• 基础截图+文本识别功能

• 集成单一AI服务（如OpenAI）

• 实现基础模板和用户选择记录

• 10个种子用户测试

V1.0阶段（3-4个月）

• 完整的端边云架构部署

• 多AI服务支持（备选方案）

• 基础模板优化算法

• 用户偏好分析报告

• 正式发布，目标100+活跃用户

V2.0阶段（6个月）

• 高级个性化引擎

• 多语言支持

• 团队协作功能

• 商业化模板市场

• 目标1000+付费用户

五、成功度量指标（KPI）

1. 用户活跃度：DAU/MAU > 30%
2. 功能使用率：截图功能使用频率 > 3次/天/用户
3. 用户满意度：NPS > 40
4. 模板优化效果：选项点击率每月提升 > 5%
5. 系统稳定性：故障恢复时间 < 15分钟

这个项目目标既包含了具体的技术指标，也明确了业务价值和阶段性成果，可以作为项目规划和评估的基准。

## 开发指南 (Development Guide)

### 1. 环境准备
- Node.js (v16+)
- npm

### 2. 后端设置 (Server)
后端使用 Node.js + Express + LowDB (本地JSON数据库)。

```bash
cd server
npm install
# 配置 .env 文件 (参考 .env.example，填入文心API Key)
npm run dev
```

### 3. 前端设置 (Client)
前端使用 Electron + React。

```bash
cd client
# 设置 Electron 镜像 (如果在中国大陆)
$env:ELECTRON_MIRROR="https://npmmirror.com/mirrors/electron/"
npm install
npm run dev
```

### 4. 项目结构
- `server/`: 后端代码
  - `src/models/`: LowDB 数据模型
  - `src/services/`: 业务逻辑 (文心API, 模板管理)
  - `src/routes/`: API 路由
  - `db.json`: 本地数据库文件 (自动生成)
- `client/`: 前端代码
  - `electron/`: Electron 主进程
  - `src/`: React 渲染进程
